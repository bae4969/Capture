# src/ — 작업 산출물 영역

에이전트가 만들어내는 **유지·재사용되는 결과물**을 저장한다. 코드·스크립트·재사용 가능한 산출물의 정식 위치.

## 규칙

1. **worker만 쓰기 가능**. orchestrator/researcher/critic은 읽기만 한다.
2. **구조는 도메인/언어별로 하위 폴더**로 분리한다 (예: `src/python/`, `src/scripts/`, `src/notebooks/`). 루트에 파일을 흩뿌리지 않는다.
3. **임시 파일·실험 산출물 금지**. 그건 `outbox/`로 간다. `src/`에 들어온 것은 "유지해야 할 자산"이라는 의미.
4. **재현 가능성**: 외부 데이터·자격증명에 의존하는 코드는 그 사실을 README나 주석에 명시한다.
5. **삭제는 사용자 확인 후**. 에이전트가 임의로 `src/`의 파일을 지우지 않는다.

## 입력 출처

- `inbox/`의 요청·자료를 받아 처리한 결과 중 **재사용 가치가 있는 것**만 `src/`로 승격한다.
- 일회성 답변·임시 분석은 `outbox/`에 머물고 `src/`로 올라오지 않는다.

## 명명

- 파일명: `snake_case.{ext}` (영문)
- 디렉토리: 도메인 명사 (`data_pipeline/`, `agents/`, `prompts/`)
- 한글 파일명은 피한다 (스크립트 호환성).
